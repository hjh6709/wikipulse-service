# WikiPulse — 추후 기능 아이디어

현재 기획서에는 없지만 추후 논의/구현을 검토할 아이디어 메모입니다.

---

## 1. 사용자 개인화 기능

로그인 사용자가 단순 조회를 넘어 플랫폼과 상호작용할 수 있게 합니다.

- **관심 토픽 구독** — 특정 키워드/카테고리 등록 시 해당 이슈 우선 노출
- **이슈 북마크** — 나중에 다시 볼 이슈 저장
- **공유** — 이슈 링크를 SNS/슬랙 등으로 공유

---

## 2. 이슈 뉴스화 가능성 예측

과거 데이터 패턴 기반으로 이슈가 주요 언론 보도로 이어질 가능성을 표시합니다.

**아이디어:**
```
편집 폭증 + Reddit 댓글 급증 + 감성 부정 70% 이상
→ 과거 유사 패턴의 73%가 24시간 내 주요 언론 보도됨
→ "뉴스로 터질 가능성 높음 ●●●○○"
```

**전제 조건:**
- 과거 이슈 데이터 충분히 쌓여야 함 (Month 3 이후)
- 실제 뉴스 보도 여부 비교 데이터 소스 필요
- 양성호(AI팀) 추가 모델 작업 필요

---

## 3. 관련 종목 연결 (경제 섹터)

이슈 키워드와 관련된 주식 종목을 보여줍니다.

**표시 방식 (안전한 표현):**
```
"Korean_War 관련 키워드 — 방산 섹터 (한화, LIG넥스원)"
"과거 유사 편집 폭증 시 해당 섹터 변동성 +23%"
```

**법적 고려사항:**
- "이 주식 오를 것 같다" 같은 투자 권유 표현 금지 → 자본시장법 라이선스 필요
- "관련 종목" + "과거 변동성 통계" 수준은 면책 조항 포함 시 가능
- Bloomberg, Reuters도 동일한 방식으로 운영 중
- **반드시 "본 정보는 투자 조언이 아닙니다" 면책 조항 필요**

**구현 방향:**
- 이슈 키워드 → 섹터 매핑 테이블 (방산, 에너지, 제약 등)
- 섹터 → 관련 종목 리스트
- 과거 유사 이슈 발생 시 해당 섹터 변동성 통계 표시

---

## 4. 비로그인 공개 페이지

트렌딩 이슈 리스트는 로그인 없이 볼 수 있게 하여 일반 사용자 유입을 유도합니다.

- `/` 랜딩 페이지 — 서비스 소개 + 실시간 트렌딩 이슈 미리보기
- 이슈 상세(감성 분석, 브리핑 등)는 로그인 후 공개
- "더 보려면 로그인" 유도 구조

---

## 5. 이슈 카테고리 분류

이슈를 카테고리로 분류하여 사용자가 원하는 분야만 필터링할 수 있게 합니다.

**카테고리 예시:** 정치, 경제, 스포츠, 연예, 사회/범죄, 과학/기술

---

## 6. 정보 제공 원칙 (전 카테고리 공통)

**핵심 원칙: WikiPulse는 데이터를 보여줄 뿐, 판단은 사용자가 한다.**

- 플랫폼이 어느 쪽이 옳다/그르다 판단 금지
- AI 브리핑은 "~라는 주장이 있습니다", "~측은 ~라고 말합니다" 형태 유지
- 양성호(AI팀)와 중립적 프롬프트 가이드라인 사전 합의 필요

**카테고리별 추가 주의사항:**

| 카테고리 | 주의사항 |
|---|---|
| 정치 | 정치 성향 편향 금지, Reddit 커뮤니티 자체 편향 가능성 명시 |
| 경제/주식 | 투자 조언 오해 방지 → "본 정보는 투자 조언이 아닙니다" 면책 조항 필수 |
| 연예 | 루머/미확인 정보 주의 → Wikipedia 편집 분쟁일 수 있음 명시 |
| 스포츠 | 상대적으로 민감도 낮음 |
| 사회/범죄 | 피의자 추정 보도 금지, 2차 피해 가능성 고려 |
| 과학/기술 | 상대적으로 민감도 낮음 |

---

## 7. 데이터 신뢰도 표시

데이터를 그냥 보여주는 것에서 나아가 신뢰도 맥락을 함께 제공합니다.

- Reddit 샘플 수가 적으면 "여론 데이터 부족" 표시
- Wikipedia 편집자가 1~2명이면 "단일 편집자 주의" 표시
- 감성 분석 score가 낮은 경우 "확신도 낮음" 표시

---

## 8. 이슈 생애주기 관리

이슈의 발생부터 소강까지 상태를 추적합니다.

**생애주기:**
```
발생 → 확산 → 정점 → 소강 → 아카이브
```

- **발생/확산/정점**: 홈 트렌딩 이슈 리스트에 노출
- **소강**: 편집 횟수가 평소 수준으로 복귀 → 홈에서 자동 제거
- **아카이브**: 히스토리 검색으로만 조회 가능

기획서의 "트렌드 아카이브" 기능과 자연스럽게 연결됩니다.

---

## 9. 이슈 상태 아이콘/이모지

이슈 생애주기 상태(발생/확산/정점/소강)를 텍스트 뱃지 대신 아이콘이나 이모지로 표현합니다.

**방향 예시:**
- 발생 🔵 / 확산 🟡 / 정점 🔴 / 소강 ⚫
- 또는 커스텀 SVG 아이콘으로 불꽃 강도 시각화

> 현재는 텍스트 뱃지로 구현, 디자인 다듬을 때 교체 예정

---

## 10. 타 언어 Wikipedia 연동 (추후 검토)

Wikipedia SSE 스트림은 전 언어 편집을 포함하므로 언어 필터 추가만으로 구현 가능합니다.

**기술적으로 가능한 것:**
- `enwiki`, `kowiki`, `jawiki` 등 언어별 필터
- 같은 이슈가 몇 개 언어 Wikipedia에서 동시에 폭증하는지 → 글로벌 이슈 판별

**전제 조건 (김찬영, 양성호 확인 필요):**
- Reddit이 영어 기반이라 비영어 이슈의 여론 데이터 부족 가능성
- DistilBERT가 영어 기반 모델 → 다국어 처리 시 별도 모델 필요
- Gemini 브리핑의 다국어 처리 가능 여부

> 영어 Wikipedia 기준으로 완성도 높인 후 Month 3 이후 검토
